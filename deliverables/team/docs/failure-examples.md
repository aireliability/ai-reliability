# Failure examples (Team)

## Refund failure (policy)
Outputs that approve refunds outside policy or imply exceptions can create billing risk.

## Tool/action failure (execution)
Outputs that claim an action happened when it didn’t (or only promise to do it later) are operationally unsafe.

## Hallucinated pricing / authority failure
Outputs that invent plans, pricing, endpoints, or “official” facts increase churn and support load.
